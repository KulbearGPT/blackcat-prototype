当前关键决策

以下口径已进入 v0.7；如发生调整，应同步更新总体规格、API、数据模型、原型和验收用例。

统一业务 API

Sapphire Discord Bot 与 Dashboard 都调用可复用 API，不在各自客户端重复实现订单、资金、返佣和权限规则。

预留后再消费

订单和订单内礼物先占用可用余额；5 分钟派单超时保持原预留，取消或 30 分钟到期才释放，避免重复占用和服务结束后追款。

自动化保留人工接管

下单、派单、取消和完成默认自动；超时、争议、礼物核对和高风险操作进入客服任务，并由满足最低等级且完成所需 step-up/MFA 的人员直接执行。

统一权限口径：权限按 L1<L2<L3<L4 累积继承，高级别自动拥有低级全部执行权限；业务操作采用最低等级直接执行，同一人可发起并执行。高额 ≥500000 必须 L4，但不强制不同人员复核；仍须保留原因、完成策略要求的 step-up/MFA，并写入不可篡改审计。
