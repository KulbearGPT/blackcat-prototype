P1-01

陪玩档案、试音与审核材料

价值
让供给质量、技能、风格和历史审核形成可追溯档案，为选择、匹配和运营质检提供可信输入。
前置
P0 陪玩身份、状态、标签、隐私分级与审计稳定；定义材料保存、授权、过期和删除政策。
风险
音频/图片隐私、内容审核、虚假材料、主观评分偏差，以及材料泄露给无权限角色。
顺序
N1 首项；先做内部审核档案，再决定哪些字段可面向用户展示。
P0 排除
Prototype 只保存准入状态与运营标签，不上传试音、作品集、公开评分或完整个人主页。
P1-02

用户选陪玩与指定陪玩

价值
把档案转化为更高意愿的下单入口，支持复购指定和有限候选选择，同时保留自动派单兜底。
前置
P1-01 可公开档案字段、可用性事实、价格版本、候选资格和指定失败后的降级规则明确。
风险
热门陪玩拥堵、选择歧视、价格/状态过期、绕过准入，以及指定失败导致订单体验变差。
顺序
N1 第二项；先支持“偏好候选 + 自动降级”，再评估强指定。
P0 排除
P0 仅按资格、状态和标签自动派单，不提供用户浏览、排行、收藏或指定陪玩。
P1-03

预约、班次与可服务时间

价值
把即时供给扩展到可计划服务，降低高峰缺人和临时爽约，支持陪玩自助维护班次。
前置
P1-01/02 的身份与选择策略、统一时区、班次容量、预留期限、改期/取消和资金释放规则。
风险
时区与夏令时、超卖、临时变更、长期资金占用、跨日服务和排班劳动合规。
顺序
N2；先单次预约和明确班次，再考虑周期排班、候补或复杂容量优化。
P0 排除
P0 订单为 IMMEDIATE_ONLY，不接受未来时间、不做班表、候补或改期。
P1-04

经营分析、财务对账与导出

价值
从最低运营统计升级为可核对的收入、退款、预留、收益、返佣、绩效和异常分析，支持标准化导出。
前置
P0 账务事实、事件时间、币种、版本、冲正链、第三方流水镜像与数据权限达到稳定口径。
风险
口径漂移、重复/迟到事件、PII 泄露、导出越权、报表与原始账不一致，以及把缓存误当账本。
顺序
N2；先对账差异和受控 CSV 导出，再增加趋势、绩效和可配置报表。
P0 排除
P0 只提供最小实时摘要和对象级时间线，不做完整 BI、月结、税务报表、批量导出或自动发报。
P1-05

多 Discord Server 与更多服务渠道

价值
让同一业务核心服务多个 Discord Server，并为其他语音/客服渠道保留一致的订单与任务入口。
前置
所有对象具备显式 guild/channel scope；身份映射、幂等、消息 Outbox、权限配置和渠道适配器边界稳定。
风险
跨 Server 数据泄露、身份碰撞、重复消息、渠道能力不对称、权限语义漂移和运维复杂度上升。
顺序
N3；先同一组织多 Discord Server，再以一个额外渠道验证适配器，不同时铺开。
P0 排除
P0 只面向单测试 Guild 与 Sapphire Bot，不做跨 Server 搜索、共享角色或其他渠道连接器。
P1-06

移动运营界面与通知中心

价值
让客服和管理人员在移动场景完成认领、核对、审批与异常响应，并统一管理待办、升级和送达状态。
前置
任务/审批 API、capabilities、step-up、深链、通知偏好、去重、已读状态和安全会话策略稳定。
风险
锁屏泄密、推送重复/延迟、移动误操作、弱网重放、会话失窃和高风险操作缺少足够上下文。
顺序
N3；先响应式任务/审批，再做站内通知中心，最后评估原生推送或轻量 App。
P0 排除
P0 仅保证 Dashboard 基本响应式与现有 Discord 通知，不做原生 App、推送编排或全渠道收件箱。

依赖与进入顺序

阶段能力必须先证明阶段退出证据
N1P1-01、P1-02P0 陪玩准入、标签、可用性、隐私权限和自动派单稳定。档案字段分级通过评审；选择/降级规则无超卖且不绕过候选资格。
N2P1-03、P1-04P0 资金预留、状态事件、冲正和第三方流水镜像能够对账。预约容量并发测试通过；样本账期差异可解释，导出权限与脱敏通过测试。
N3P1-05、P1-06scope、Actor Context、Outbox、任务/审批和 step-up 在单 Guild 上稳定。跨 Server 隔离 E2E 通过；移动端关键操作在弱网和重复通知下保持幂等。

候选能力潜在价值重新评估触发条件主要约束
微信 / 邮箱认证扩大非 Discord 用户与工作人员入口。出现稳定的非 Discord 获客或后台账号需求。账号合并、验证安全、隐私同意与身份恢复。
白标、定制域名与可配置组件支持品牌化交付和不同团队的界面差异。至少两个付费组织需要独立品牌且核心流程一致。主题隔离、域名证书、组件兼容和升级成本。
多租户 / 多门店 / 分支机构支持组织级隔离、分店运营和集团视图。单组织多 Server 已稳定，且真实客户要求组织级结算。租户隔离、密钥、账务、权限继承与数据迁移。
连接器市场以标准适配器扩展客服、支付、语音和数据工具。至少三个重复渠道需求且适配器契约成熟。权限授权、密钥托管、版本兼容和第三方 SLA。
营销自动化、VIP、优惠券与活动促进复购、分层运营和活动转化。核心留存、毛利和风控口径稳定，并有可测量的增长假设。滥用、补贴核算、隐私、规则爆炸和用户公平性。

下一阶段立项门槛

P0 证据
P0 blocker 全部通过，并有真实运行周期的订单、预留、派单、任务和异常数据。
问题证据
每个 P1 项目必须绑定可量化问题，不因竞品存在某功能而直接复制。
范围重签
立项时重新产出 UI、OpenAPI、数据迁移、权限、配置、Backlog 和验收；本页不能代替规格。
安全评审
涉及档案、音频、导出、多渠道、推送或新身份时，先完成隐私、授权和数据保留评审。
顺序纪律
按依赖推进；不得为 Nice to Have 提前引入多租户、营销或连接器复杂度。
